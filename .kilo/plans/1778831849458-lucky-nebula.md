# Fix React Error #130 - LazyIcon Component

## Problem
The deployed application is still showing React error #130:
```
Error: Minified React error #130; visit https://reactjs.org/docs/error-decoder.html?invariant=130
```

This error means "Element type is invalid: expected a string (for built-in components) or a class/function (for composite components), but got: object"

## Root Cause Analysis
1. The deployed version hasn't been updated with the LazyIcon fix
2. The `LazyIcon` component uses `React.lazy()` with a dynamic import that may return an object instead of a Promise
3. The `getIconNameForApp` function may return icon names that don't match the icon map keys

## Implementation Plan

### Step 1: Fix LazyIcon Component (CRITICAL)
The `React.lazy()` must be called at module level, not inside a component. Move it outside:

```tsx
// Create lazy components at module level for stability
const createLazyIcon = (importFunc: () => Promise<{ default: ComponentType<any> }>) => {
  return React.lazy(importFunc);
};

// Icon mapping with pre-created lazy components
const iconMap: Record<string, ComponentType<any>> = {
  "bar-chart-2": createLazyIcon(() => import("lucide-react").then((m) => ({ default: m.BarChart2 }))),
  // ... rest of icons
};

const LazyIcon: React.FC<LazyIconProps> = ({ name, className = "h-6 w-6", size }) => {
  const iconName = typeof name === "string" ? name : "sparkles";
  const IconComponent = iconMap[iconName] || FallbackIcon;
  
  return (
    <Suspense fallback={<FallbackIcon className={className} size={size} />}>
      <IconComponent className={className} size={size} />
    </Suspense>
  );
};
```

### Step 2: Add Validation

- Ensure fallback is always used for invalid icons
