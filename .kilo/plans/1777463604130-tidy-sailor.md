## Superpower Skills Strategy

### Phase 1: Build Conversion Tooling (1-2 days)

**Objective:** Create automated converters that transform Streamlit Python → Netlify TypeScript + React.

#### Task 1.1: App Analyzer Script

**File:** `scripts/analyze-streamlit-app.js` (or `.ts`)

**Purpose:** Parse each Streamlit app and extract structured metadata.

**Output per app:**
```json
{
  "appName": "ai_reasoning_agent",
  "mainFile": "starter_ai_agents/ai_reasoning_agent/local_ai_reasoning_agent.py",
  "category": "starter_ai_agents",
  "uiType": "chat" | "form" | "multi-tab" | "file-upload" | "dashboard",
  "inputFields": [
    {"name": "question", "type": "text", "required": true},
    {"name": "model", "type": "select", "options": ["gpt-4", "gpt-3.5"]}
  ],
  "outputFormat": "markdown" | "json" | "html" | "visualization",
  "aiProvider": "openai" | "anthropic" | "google" | "agno" | "ollama",
  "model": "gpt-4o-mini",
  "usesTools": ["web_search", "code_execution", "file_processing"],
  "externalServices": ["supabase", "firecrawl", "exa", "qdrant"],
  "hasSessionState": true/false,
  "hasHistory": true/false,
  "complexity": "simple" | "multi-agent" | "pipeline" | "rag"
}
```

**Implementation:**
- AST parsing of Python files (use `@typescript-esno/esno-parser` or simple regex for key patterns)
- Detect Streamlit patterns: `st.text_input`, `st.button`, `st.chat_input`, `st.file_uploader`, `st.tabs`, `st.session_state`
- Extract OpenAI/Anthropic client initialization
- Identify agent framework usage (agno, crewai, langchain, openai-agents)
- Generate structured JSON catalog (extend existing `streamlit_apps_catalog.json`)

#### Task 1.2: Function Generator

**File:** `scripts/generate-netlify-function.ts`

**Purpose:** Convert Streamlit app logic into Netlify Function handler.

**Template-based generation:**
- Template A: Single-call LLM agent (simple)
- Template B: Multi-step pipeline (progress stages)
- Template C: Agent team/orchestration (parallel calls)
- Template D: RAG with vector search (DB queries)
- Template E: Streaming responses (async SSE)

**Process:**
1. Read app analyzer JSON
2. Match to function template
3. Inject:
   - Input/output TypeScript interfaces
   - AI provider SDK initialization (switch environment variables)
   - Core logic translated from Python → TypeScript (LLM calls preserved)
   - Supabase logging (always insert to `ai_agent_runs`)
   - Error handling + CORS
4. Write to `netlify/functions/{slug}.ts`
5. Update `netlify/functions/index.ts` export

**Complexity Mapping:**
- `starter_ai_agents/` → Template A (single LLM call)
- `rag_tutorials/` → Template D (vector DB + LLM)
- `advanced_ai_agents/multi_agent_apps/` → Template C (orchestrator)
- `voice_ai_agents/` → Special (audio processing skipped or stubbed)
- `mcp_ai_agents/` → Template C (MCP tool calling)

#### Task 1.3: React Component Generator

**File:** `scripts/generate-react-component.ts`

**Purpose:** Generate React page component from Streamlit UI structure.

**Template variants:**
- **Simple Form Page:** Single text input + button → results
- **Multi-Tab Page:** Input tab + results tabs (overview, details, raw data)
- **Chat Interface:** Message history + streaming response
- **Dashboard Page:** Multiple widgets + filters + data table

**Generated structure:**
```
src/pages/agents/{
  AppNamePage.tsx          # Main page component
  AppNameForm.tsx          # Input form (extracted)
  AppNameResults.tsx       # Results display (extracted)
  AppNameHistory.tsx       # Optional history view
}
```

**Implementation:**
- Based on `uiType` from analyzer
- Use common UI components: `Input`, `Textarea`, `Button`, `Select`, `Tabs`
- Include loading states, error handling, auth check
- Auto-wire to fetch(`/.netlify/functions/{slug}`)
- Add result export (copy, download, share)

#### Task 1.4: Route & Data Auto-Registration

**Script:** `scripts/register-apps.ts`

**Purpose:** Automatically add routes and app metadata to the main app registry.

**Operations:**
1. Read all generated app configs from `scripts/generated-apps/`
2. Update `src/App.tsx`:
   - Add `import` statement for each new page
   - Add `<Route path="/agent/{slug}" element={<Page />} />`
3. Update `src/data/appsData.ts`:
   - Add app entry with name, slug, description, icon, category, gradient
4. Update `src/components/AgentCard.tsx` (if new category)
5. Generate sales/marketing copy from app description (use AI or templates)

**Smart Grouping:**
- Group similar apps into collections (e.g., "Research Agents", "RAG Tools", "Voice AI")
- Order by popularity/complexity (starter → advanced)
- Auto-assign to Tools Hub categories

## Phase 2: Batch Conversion Execution (2-3 days)

### Day 1: Convert Simple Apps (40 starter + voice agents)

**Batch size:** 16 starter + 3 voice = 19 apps

**Automation flow per app:**
```bash
# For each app in streamlit_apps_catalog.json:
1. node scripts/analyze-streamlit-app.js → analyze output/{app}.json
2. node scripts/generate-netlify-function.ts → netlify/functions/{slug}.ts
3. node scripts/generate-react-component.ts → src/pages/agents/{Slug}Page.tsx
4. node scripts/register-apps.ts → update App.tsx + appsData.ts
5. npm run build → verify no TypeScript errors
```

**Human review required:** Spot-check 5 random apps (25% sample) for correctness.

**Batch processing script:** `scripts/batch-convert-all.sh` (or `.mjs`)
- Parallel processing: run 4-5 conversions concurrently
- Logging: capture errors per app, continue on failure
- Summary report: count of successes/failures, common error patterns

### Day 2: Convert RAG Agents (20 apps)

**Considerations:**
- RAG apps need Qdrant/vector DB integration already in Supabase
- Most use local embeddings or Ollama → need to map to hosted vector DB
- Some use pgvector already → easier conversion
- Auto-generate DB schema migration for Qdrant collection setup (if needed)

**Special handling:**
- Apps using local files (PDF uploads) → need file upload → Supabase Storage integration
- Apps using ChromaDB/Qdrant local → map to Supabase remote
- Generate vector search functions if needed (reuse existing patterns)

### Day 3: Convert Advanced & Multi-Agent Apps (37+ apps)

**Considerations:**
- Most complex (25+ use external APIs: Firecrawl, Exa, Together, etc.)
- Multi-agent orchestration: convert Python async agents → TypeScript parallel calls
- Agent frameworks (CrewAI, AutoGen, Agno, Phidata) → need framework-specific adapters

**Strategy:**
- Group by framework:
  - **agno-based** (most common): Convert to sequential OpenAI calls with tool calling
  - **creai-based**: Map CrewAI agents → independent Netlify function calls
  - **autogen-based**: Map AutoGen agents → parallel function invocations
  - **langchain-based**: Convert chains → single function with step-by-step execution
- Some apps may need simplification (browser automation not possible serverless)

**Fallback:** For apps requiring long-running processes (>10s), mark as "async job" → generate job-queue pattern with webhook notification.

### Day 4: Convert MCP & Framework Tutorials (16 apps)

**MCP Agents:**
- MCP (Model Context Protocol) integration needs MCP server → still possible in Netlify Functions with SSE
- Convert MCP tool calls to direct API calls where possible
- Or stub out MCP calls, show "Not available in hosted version" message

**Framework Tutorials:**
- Mostly simple demos → easy conversion
- Many use Google ADK, Ollama → map to OpenAI
- Remove local-only dependencies (gradio, uvicorn)

## Phase 3: Integration & Deployment Polish (1 day)

### Task 3.1: Environment Configuration

**Required API keys:**
- `OPENAI_API_KEY` (required for most)
- `ANTHROPIC_API_KEY` (for Claude-based agents)
- `GOOGLE_GENERATIVE_AI_KEY` (for Gemini agents) — optional
- `EXA_API_KEY` (for web-search agents)
- `FIRECRAWL_API_KEY` (for Firecrawl-based agents)
- `TOGETHER_API_KEY` (for Together-based agents)

**Strategy:** Centralize all keys in Netlify UI environment variables. Document which agents need which keys.

**Graceful degradation:** If key missing, show helpful error message + upgrade prompt.

### Task 3.2: Database Schema Extensions

**Existing tables:**
- `ai_agent_runs` (already exists, sufficient)
- `apps` (already exists)

**Potential additions:**
- `agent_favorites` (user -> favorite agents)
- `agent_ratings` (user ratings/reviews)
- `agent_template_configs` (save customized agent configs)

**Complex/Long-Running Agent Handling:**
For agents that exceed Netlify/Supabase function timeout (10-60s) or require hours:
- Implement async job queue pattern:
  1. Function enqueues job in `agent_jobs` table (status: pending)
  2. Returns `job_id` immediately to frontend
  3. Background worker (Supabase Edge Function with cron) processes jobs
  4. Frontend polls job status or receives webhook/email on completion
- Database table `agent_jobs`: `id, agent_type, user_id, input_data, status, result_data, created_at, completed_at`
- Frontend shows "Processing… Check back later" with progress indicator

**Simplification for Non-Serverless Apps (Option C):**
Browser automation (Playwright/browser-use), GPU training, video rendering → replace with simplified serverless alternative:
- Speech trainer → audio analysis only (skip video lip-sync)
- Video generation → static image or short clip only
- Browser automation → "Download Desktop App" or simplified web-only version
- Document upscaling → lower resolution serverless version

Mark simplified versions in spec with "Cloud Lite" badge.

### Task 3.3: Thumbnail Generation

**Use existing scripts:**
- `scripts/generate-app-thumbnails.js` → run after all apps registered
- Auto-generate unique SVG thumbnails per agent
- Upload to Supabase Storage `app-assets/thumbnails/`

### Task 3.4: Testing & QA

**Automated tests:**
- Build verification: `npm run build` (Vite TypeScript build)
- Lint check: `npm run lint`
- Type checking: `npm run typecheck`
- E2E smoke test: Playwright script to visit each route, verify form renders, submit test data

**QA checklist per app category:**
- [ ] Form renders without errors
- [ ] Submits to Netlify function without CORS errors
- [ ] Displays results correctly
- [ ] Saves to `ai_agent_runs` table
- [ ] Handles API key missing gracefully
- [ ] Mobile responsive layout
- [ ] Loading states visible
- [ ] Error messages user-friendly

### Task 3.5: Netlify Functions Deployment Automation

**Current gap:** Netlify Functions aren't auto-deployed in build pipeline.

**Solution A (Recommended):** Use Netlify CLI
```bash
# Add to package.json:
"deploy:functions": "netlify deploy --functions --prod"

# Or use Netlify Build plugin for automatic function bundling
```

**Solution B:** Deploy as Edge Functions via Supabase (already used for admin APIs)
- Move all agent functions to Supabase Edge Functions (Deno)
- Netlify frontend calls `https://<project>.functions.supabase.co/agent-name`
- Benefits: unified auth, no cold-start issues, env var management

**Recommendation:** Use Solution B (Supabase Edge Functions) for:
- Simpler deployment (already automated via `DEPLOY_FUNCTIONS_LOCAL.sh`)
- Unified auth (Supabase JWT)
- No Netlify Functions cold-start delays
- Single platform management

**Action:** Convert Netlify function syntax to Deno-compatible (minor changes).

## Risk Mitigation

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Complex multi-agent apps fail conversion | Medium | High | Fallback: simplify to single-agent version, keep complex ones as premium features |
| External API dependencies break | High | Medium | Stub missing APIs with mock responses, show "Coming Soon" |
| TypeScript build errors from generated code | High | Medium | Compile after each batch, fix templates iteratively |
| Netlify Functions timeout (10s default) | Medium | High | Batch async processing, use Supabase DB for long jobs, webhook notifications |
| RAG vector DB not available | Low | Medium | Use Supabase pgvector, pre-migrate Qdrant collections |
| License/copyright issues from original apps | Low | High | Only use for internal commercial use, attribute original source |
| App UI looks too similar (template fatigue) | Medium | Low | Introduce 3-4 visual theme variants (gradient packs) |
| Some Streamlit APIs have no React equivalent | Medium | Low | Skip unsupported features (st.download_button → custom download link) |
