## Goals

## Steps

### 1. Fix Failed Migration
- Migration `20260423190312_add_super_admin_role_for_dean.sql` failed because user_id doesn't exist
- Use `supabase_execute_sql` to check if user exists in auth.users
- If user doesn't exist, either create user via auth or modify migration to handle missing user
- Use `supabase_apply_migration` to retry the specific migration after fixing

### 2. Enable RLS on All Tables
- Use `supabase_get_advisors` to get the exact list of 180 tables needing RLS enabled
- Use `supabase_execute_sql` to run `ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;` for each table
- Batch the SQL commands to avoid individual calls

### 3. Add Missing Foreign Key Indexes
- Use `supabase_get_advisors` to get the exact list of 262 unindexed foreign keys
- Use `supabase_execute_sql` to run `CREATE INDEX CONCURRENTLY idx_table_column ON table_name(column_name);` for each
- Batch index creation to optimize performance

### 4. Verify Schema Completeness
- Compare current schema with requirements in `create-database-schema.test.ts`
- Ensure all required tables, policies, and functions exist
- Use `supabase db diff` to check for differences

### 5. Test Application Functionality
- Run the application and test database operations
- Verify CRUD operations work with RLS
- Check that auth and data fetching work properly

- `supabase_apply_migration` - Apply specific migrations that failed
- `supabase_list_tables` - Verify table existence and structure
- `supabase db diff` - Check current schema vs desired state
- `supabase db reset` - Reset database if major issues require clean slate
