# App Payment Enhancement Plan

## 🎯 **Goal**
Enhance the existing app purchase system to support individual app purchases ($37) and all-apps bundle ($597) with lifetime access, while maintaining the current GTM modal → purchase modal flow.

## 📋 **Current State Analysis**

### ✅ **What's Already Working:**
- Stripe integration with API keys configured
- Existing PurchaseModal component for app purchases
- Supabase edge function for checkout sessions
- Rich app data with pricing, features, descriptions
- ~138 apps in the current catalog (not 1900+ Streamlit apps)
- ProductDetailModal shows GTM content when clicking apps
- Sequential modal flow: GTM modal → Purchase modal

### 🔍 **Key Findings:**
- Apps have detailed metadata (pricing, features, categories)
- Existing payment flow: PurchaseModal → Supabase Edge Function → Stripe Checkout
- Apps categorized by functionality (AI agents, tools, utilities)
- Current pricing model supports one-time payments

### **Database Schema Updates:**
**Use existing tables, add purchase type tracking:**
```sql
-- Existing purchases table enhancement
ALTER TABLE purchases ADD COLUMN IF NOT EXISTS purchase_type TEXT DEFAULT 'single';
ALTER TABLE purchases ADD COLUMN IF NOT EXISTS is_bundle BOOLEAN DEFAULT false;

-- Add app access tracking (if needed)
CREATE TABLE IF NOT EXISTS user_app_access (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  app_id TEXT NOT NULL, -- References appsData.ts app IDs
  access_granted BOOLEAN DEFAULT true,
  purchase_id UUID REFERENCES purchases(id),
  granted_at TIMESTAMP DEFAULT NOW()
);

-- Add bundle purchases tracking
CREATE TABLE IF NOT EXISTS bundle_purchases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  purchase_id UUID REFERENCES purchases(id),
  bundle_type TEXT DEFAULT 'all_apps', -- 'all_apps' for $597 bundle
  access_granted_at TIMESTAMP DEFAULT NOW()
);
```
