interface ExtendedSalesCopy {

    quote: string;          // 1-2 sentences

#### Phase 0.2: Supabase Edge Functions Architecture Design

**Superpowers Skills Approach:** Use different tonalities/agent specializations per function category to optimize migration quality.

**Architecture Decision:**
```
┌─────────────────────────────────────────────────────────────┐
│  Frontend (React)                                          │
│  fetch('/.netlify/functions/x')    →    fetch('/functions/x')│
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  Supabase Edge Functions (Deno Runtime)                    │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ Function: ai-blog-to-podcast-agent                   │ │
│  │ - Deno handler (TypeScript)                          │ │
│  │ - Imports: openai SDK (Deno version), agno, Playwright│ │
│  │ - Secrets: OPENAI_API_KEY from Supabase config       │ │
│  │ - Returns: JSON response to frontend                 │ │
│  └───────────────────────────────────────────────────────┘ │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ Function: chat-with-pdf (RAG)                        │ │
│  │ - Deno handler                                        │ │
│  │ - Supabase pgvector query (same project)             │ │
│  │ - OpenAI embeddings + chat completions               │ │
│  └───────────────────────────────────────────────────────┘ │
│  ... (111+ functions)                                     │
└─────────────────────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  External APIs (OpenAI, Google, etc.)                      │
│  API keys stored as Supabase Secrets (encrypted at rest)   │
└─────────────────────────────────────────────────────────────┘
```

**Key Design Choices:**

1. **Runtime:** Deno (Supabase Edge Functions default) — compatible with many Node packages via npm:deno shims or Deno porting
2. **Secrets Management:** Supabase Dashboard → Project Settings → Secrets (migrate from Netlify env vars)
3. **Database:** Use existing Supabase Postgres + pgvector (RAG apps already reference it)
4. **Cold Start Mitigation:** Keep functions lightweight; use `--no-config` flag; pre-warm critical functions if needed
5. **Timeouts:** Edge Functions have 60s limit — long-running agents → async job queue (Phase 2 enhancement)

**Superpowers Skills Tonality Mapping for Migration:**
- **Starter Agents** (Jobs + Ogilvy): Fast, valuable MVP delivery — "ship daily"
- **RAG Apps** (Bezos + Trusted Advisor): Reliability, accuracy, cite sources — "customer-obsessed retrieval"

##### Sub-Agent 0.4.2: RAG Applications (20 functions)
**Tonality:** Bezos + Trusted Advisor — Reliable, cited, accurate
**Pattern:** Embeddings + Vector DB + Retrieval + Generation
**Special Considerations:** Need vector DB. Use Supabase pgvector (already available).

**Migration Template:**
```typescript
// supabase/functions/chat-with-pdf/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";
import OpenAI from "npm:openai";

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')! // service role for DB access
);
const openai = new OpenAI({ apiKey: Deno.env.get('OPENAI_API_KEY') });

serve(async (req) => {
  const { question, user_id } = await req.json();

  // 1. Embed question
  const embedding = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: question
  });

  // 2. Query pgvector (Supabase RPC or direct SQL)
  const { data: docs } = await supabase.rpc('match_documents', {
    query_embedding: embedding.data[0].embedding,
    match_threshold: 0.78,
    match_count: 5
  });

  // 3. Construct prompt with context
  const context = docs.map(d => d.content).join('\n\n');

  // 4. Call chat completion
  const completion = await openai.chat.completions.create({
    model: "gpt-4-turbo",
    messages: [
      { role: "system", content: "Answer based on the provided context..." },
      { role: "user", content: `Context: ${context}\n\nQuestion: ${question}` }
    ]
  });

  return new Response(JSON.stringify({
    answer: completion.choices[0].message.content,
    sources: docs.map(d => ({ title: d.title, url: d.url }))
  }));
});
```

**RAG-Specific Tasks:**
- Ensure `vector` extension enabled: `CREATE EXTENSION vector;` (likely already done)
- Verify `documents` table exists with `embedding vector(1536)` column
- Create RPC function `match_documents` if not exists (SQL: `SELECT * FROM documents WHERE embedding <-> query_embedding < 1 - match_threshold`)
- For local RAG variants (Llama, Deepseek): call local Ollama instance via HTTP (running separately) or use OpenAI as fallback

##### Sub-Agent 0.4.3: Voice AI Agents (3 functions)
**Tonality:** Chris Voss + Socratic — Empathetic, clear, active listening
**Pattern:** STT (speech-to-text) → LLM processing → TTS (text-to-speech)
**Realtime variant:** WebSocket connection (Edge Functions support websockets? Check Supabase docs — may need separate service)

**Migration Template (Non-realtime):**
```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import OpenAI from "npm:openai";
import { encode } from "https://deno.land/std@0.168.0/encoding/base64.ts";

const openai = new OpenAI({ apiKey: Deno.env.get('OPENAI_API_KEY') });

serve(async (req) => {
  const { audio_base64, user_id } = await req.json();

  // 1. Decode audio (webm/wav/mp3)
  const audioBuffer = Uint8Array.from(atob(audio_base64), c => c.charCodeAt(0));

  // 2. Transcribe (Whisper)
  const transcription = await openai.audio.transcriptions.create({
    file: new File([audioBuffer], "audio.webm", { type: "audio/webm" }),
    model: "whisper-1"
  });

  // 3. Process with LLM (agent logic)
  const completion = await openai.chat.completions.create({
    model: "gpt-4-turbo",
    messages: [
      { role: "system", content: "You are a helpful customer support agent..." },
      { role: "user", content: transcription.text }
    ]
  });

  const responseText = completion.choices[0].message.content;

  // 4. TTS (optional)
  const tts = await openai.audio.speech.create({
    model: "tts-1",
    voice: "alloy",
    input: responseText
  });
  const audioArrayBuffer = await tts.arrayBuffer();
  const audioBase64 = btoa(
    new Uint8Array(audioArrayBuffer).reduce((data, byte) => data + String.fromCharCode(byte), '')
  );

  return new Response(JSON.stringify({
    transcription: transcription.text,
    response: responseText,
    audio_response: audioBase64
  }));
});
```

**Note:** Realtime voice (WebSocket streaming) may exceed Edge Function 60s limit or require persistent connection. Consider:
- Option A: Use Supabase Realtime channel + separate Deno process (not Edge Function)
- Option B: Use dedicated voice service (Twilio, Daily.co) and call Edge Function only for LLM inference
- For MVP, implement non-realtime (upload → process → download)

##### Sub-Agent 0.4.4: Advanced Multi-Agent Teams (10+ functions)
**Tonality:** Challenger + Growth Hacker — Orchestrate, scale, disrupt
**Pattern:** OpenAI Agents SDK handoffs + tool calling + memory

**Key Challenge:** Edge Functions are stateless — agent teams maintain conversation state via database. Need `agent_runs` table:

```sql
CREATE TABLE agent_runs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  agent_team VARCHAR(255),
  conversation_id UUID,
  message_history JSONB DEFAULT '[]',
  current_agent VARCHAR(100),
  status VARCHAR(50) DEFAULT 'running',
  result JSONB,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

**Migration Template (Agents SDK):**
```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import OpenAI from "npm:openai";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const openai = new OpenAI({ apiKey: Deno.env.get('OPENAI_API_KEY') });
const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

// Agent definitions (port from Python/Node Agents SDK)
const agents = {
  researcher: { name: "Researcher", tools: [web_search, browse_url] },
  analyst: { name: "Analyst", tools: [analyze_data, create_chart] },
  writer: { name: "Writer", tools: [draft_report] }
};

serve(async (req) => {
  const { user_id, message, conversation_id } = await req.json();

  // Load or create conversation state
  let run = await supabase
    .from('agent_runs')
    .select('*')
    .eq('id', conversation_id)
    .single();

  if (!run) {
    run = await supabase.from('agent_runs').insert({
      user_id, agent_team: 'deep_research', message_history: []
    }).select().single();
  }

  // Append user message
  const history = [...run.message_history, { role: 'user', content: message }];

  // Orchestrate agents (handoff loop)
  let currentAgent = 'researcher';
  let maxSteps = 5;
  while (maxSteps-- > 0) {
    const agent = agents[currentAgent];
    const response = await openai.chat.completions.create({
      model: "gpt-4-turbo",
      messages: history,
      tools: agent.tools,
      tool_choice: "auto"
    });

    const msg = response.choices[0].message;
    history.push(msg);

    if (msg.finish_reason === 'stop') {
      break;
    }

    // Handle tool calls → update currentAgent via handoff
    currentAgent = determineNextAgent(msg, currentAgent);
  }

  // Save updated history
  await supabase.from('agent_runs').update({
    message_history: history,
    updated_at: new Date().toISOString()
  }).eq('id', run.id);

  return new Response(JSON.stringify({ history }));
});
```

**Complexities to handle:**
- Tool calling → need Deno-compatible SDK for Playwright, Firecrawl, browser-use
  - Options: call external microservice; use HTTP APIs (Firecrawl API, Browserbase API)
  - Or bundle browser automation in function (heavy, may exceed limits)
- Memory: Store conversation in Supabase JSONB column; load on each request
- Long-running: If agent loop exceeds 60s, switch to async job pattern → return `jobId` → poll for status

| Deno compatibility: npm packages missing | Medium | Use npm: specifier for compatible packages; fallback to HTTP API calls; create shims |
| Secrets not migrating (missing keys) | High | Comprehensive audit of Netlify env + code; test each function immediately after deploy |
| Vector DB (pgvector) not optimized | Medium | Create indexes BEFORE migration; test RAG query performance; add connection pooling |
| Rate limits (OpenAI, other APIs) | Medium | Implement per-user rate limiting in Edge Function; queue/backoff; monitor usage daily |
| Functions exceed Supabase plan limits (compute, bandwidth) | Medium | Check Supabase Pro limits; upgrade if needed; optimize payload sizes |
| Browser automation (Playwright) won't run in Edge | High | Use external service (Browserbase, Firecrawl API) instead of in-function Playwright |
