# Authentication Fix Plan: Remove Admin Requirement for Dashboard Access

## Problem Analysis
The application is currently logging users out immediately after login because the authentication flow incorrectly requires admin privileges for users to stay on the dashboard. Regular users need to access the dashboard without admin roles.

## Root Cause
- Users may not have the 'user' role assigned in the user_roles table
- Potential admin-only checks in dashboard components or hooks
- RLS policies may be blocking access for non-admin users
- Auth state changes may be triggering logout due to access failures

## Solution Overview
Remove admin requirements from the authentication flow and ensure regular users can access the dashboard. This involves:
1. Ensuring all users have appropriate roles assigned
2. Verifying dashboard components don't require admin access
3. Updating RLS policies if needed
4. Testing the authentication flow for regular users

## Implementation Steps

### 1. Ensure User Roles are Assigned
- [ ] Run the apply-auth-fixes edge function to create user_roles entries for all existing users
- [ ] Verify the handle_new_user trigger creates 'user' role for new signups
- [ ] Check that all users have 'user' role in user_roles table

### 2. Verify Dashboard Access Requirements
- [ ] Confirm DashboardPage.tsx doesn't have admin role checks
- [ ] Check useUserStats hook doesn't require admin permissions
- [ ] Check useUserAccess hook works for regular users
- [ ] Verify ProtectedRoute only checks authentication, not roles

### 3. Update RLS Policies if Needed
- [ ] Ensure user_app_access table allows users to read their own records
- [ ] Ensure videos table allows users to read their own videos
- [ ] Confirm current_user_id() and is_admin() functions work correctly

### 4. Test Authentication Flow
- [ ] Test signup creates user with 'user' role
- [ ] Test login doesn't immediately logout regular users
- [ ] Test dashboard loads successfully for regular users
- [ ] Verify no auth state changes cause premature logout

### 5. Fix Any Identified Issues
- [ ] Remove any admin-only checks from dashboard components
- [ ] Update auth error handling to not logout on access failures
- [ ] Ensure session management works for regular users

## Files to Check/Modify
- `src/pages/DashboardPage.tsx` - Main dashboard component

- Dashboard loads without admin requirements
- No immediate logout after successful login
- User stats and access data load correctly
- All auth state changes work properly for regular users
