## Objective
Test that a user can successfully login and logout using the authenticated Supabase connection (with the "Invalid API key" error now resolved).

## Background
- The "Invalid API key" error was caused by missing `.env` file with valid Supabase credentials
- `.env` file has been created with project `bzxohkrxcwodllketcpz` and valid anon key
- Supabase connection verified: signup and signin endpoints return valid responses
- Vite dev server running at `http://localhost:5173/`
- **NO email authentication required**: Supabase setting `mailer_autoconfirm: true` (verified via `/auth/v1/settings` endpoint)
- Users get instant access after signup - no email verification needed

## Test Strategy
Use Node.js with Supabase JS client to test authentication flows via API (browser testing unavailable in container environment).

## Steps

### 1. Verify Environment Configuration
- [ ] Confirm `.env` file exists with correct `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`
- [ ] Run `npm run validate-env` to confirm Supabase configuration is correct
- [ ] Verify dev server is running on `http://localhost:5173/`

### 2. Verify No-Email-Authentication Configuration
- [ ] Confirm Supabase dashboard setting: `mailer_autoconfirm: true` (verified via `/auth/v1/settings` endpoint)
- [ ] Check `src/context/AuthContext.tsx` signUp function has `emailConfirm: false` in options
- [ ] Verify `supabase/config.toml` has `enable_confirmations = false` under `[auth.email]` (if not, add it)
- [ ] Confirm users get instant access after signup (no email verification required)

### 3. Create Test User
- [ ] Use existing test file approach from `test-signup-login.mjs`
- [ ] Create test user with email `test-login-logout-${timestamp}@example.com`
- [ ] Use password: `TestPassword123!`
- [ ] Verify user creation returns valid `user.id` and `session.access_token`
- [ ] Confirm `email_confirmed_at` is set immediately (instant access)

### 4. Test Login Flow
- [ ] Use Supabase anon client: `createClient(url, anonKey)`
- [ ] Call `supabase.auth.signInWithPassword()` with test credentials
- [ ] **Verify login success:**
  - `data.user` is not null
  - `data.session` is not null
  - `data.session.access_token` exists
  - `data.session.expires_in` is valid (3600 seconds)
  - `data.user.email` matches test email
- [ ] Log user details: `user.id`, `user.email`, `email_confirmed_at`

### 5. Test Session Persistence
- [ ] After login, call `supabase.auth.getSession()`
- [ ] **Verify session persists:**
  - `data.session` is not null
  - `data.session.user.id` matches logged in user
- [ ] Wait 2 seconds, call `getSession()` again to verify session remains

### 6. Test Logout Flow
- [ ] Call `supabase.auth.signOut()`
- [ ] **Verify logout success:**
  - No error returned
  - Call `supabase.auth.getSession()` again
  - `data.session` should be null
- [ ] Log confirmation: "✅ Signed out successfully"

### 7. Test Login After Logout
- [ ] Attempt login again with same test credentials
- [ ] **Verify can login again:**
  - `data.user` is not null
  - `data.session` is not null
  - New `access_token` is different from previous

### 8. Test Invalid Credentials Handling
- [ ] Attempt login with wrong password
- [ ] **Verify error handling:**
  - `error` is not null
  - `error.message` contains "Invalid login credentials"
  - `data.user` is null

### 9. Cleanup Test User
- [ ] Use admin client with service role key
- [ ] Delete test user: `adminClient.auth.admin.deleteUser(userId)`
- [ ] Verify deletion: `{}` response (success)

### 10. Frontend Integration Check (Optional)
- [ ] Verify `src/utils/supabase.ts` loads env vars correctly
- [ ] Confirm `AuthContext.tsx` signIn function calls `supabase.auth.signInWithPassword()`
