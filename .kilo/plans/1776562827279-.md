# Plan: Fix Supabase User Import Issue

## Problem Analysis
The CSV import system expects files with columns: `Customer Name`, `Customer Email`, `Campaign`, `Product`

But the user's CSV files have columns: `first_name`, `last_name`, `email`, `total_spend`, `total_orders`, `last_purchase`, `segment`, `products_purchased`

When the backend parses the CSV, it incorrectly maps:
- Customer Name = first_name
- Customer Email = last_name âťŚ (should be email)
- Campaign = email âťŚ (should be something else)
- Product = total_spend âťŚ (should be from products_purchased)

This causes import failures because emails are treated as campaigns and actual email addresses are lost.

## Solution Options

### Option 1: Transform CSV to Expected Format (Recommended - Quick Fix)
Create a script to transform the user's CSV format to match what the import system expects.

### Option 2: Update Import System
Modify the CSV import function to handle the user's specific CSV format with proper column mapping.
